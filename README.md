# Market Announcements Suite

A 3-part toolkit for pulling Indian stock-exchange corporate announcements
(BSE Equity, BSE SME, NSE Equity, NSE SME), scoring them for "investment
ideas," and browsing everything in a Streamlit dashboard.

```
┌──────────────────────────┐     ┌──────────────────────────────┐     ┌───────────────────────┐
│ market_announcements.py  │ ──▶ │ announcement_ideas_pipeline.py│ ──▶ │  bse_market_suite.py   │
│  scrapes BSE/NSE          │     │  scores BSE Equity rows for   │     │  Streamlit dashboard:  │
│  → writes 4 SQLite DBs    │     │  "idea" categories             │     │  browse/search/charts/ │
│                            │     │  (uses idea_rules.py)         │     │  AI digest             │
└──────────────────────────┘     └──────────────────────────────┘     └───────────────────────┘
```

## What each file does

| File | Role |
|---|---|
| **`market_announcements.py`** | Standalone CLI scraper. Fetches announcements from 4 sources and stores each in its own SQLite database. Also does queries, text reports, CSV export, stats, and an unattended `schedule` mode for cron. |
| **`announcement_ideas_pipeline.py`** | Reads `bse_equity.db` (written by the scraper above) and layers a rule-based "idea board" on top of it — scoring each announcement (0–100) against categories like *Acquisition*, *Dividend*, *Capex*, etc. Writes results back into the same DB (new tables only; doesn't touch the scraper's tables). |
| **`idea_rules.py`** | The keyword taxonomy + weights that `announcement_ideas_pipeline.py` scores against (12 idea types across 6 groups — Deal Activity, Capital Actions, Leadership & Governance, Performance & Ratings, Regulatory & Risk, Growth & Expansion). Edit this to tune the scoring. *(This file wasn't in the original upload — the pipeline imports it but it was missing, so it's included here so the pipeline actually runs.)* |
| **`bse_market_suite.py`** | The Streamlit dashboard. Reads all 4 databases (read-only) plus the idea-scores tables. Has login (its own `market_users.db`, auto-created), search/filter, charts, keyword/trigger/cluster insights, an optional Claude-powered "AI Digest," and an embedded Idea Board tab under BSE Equity. |

### Databases (all SQLite, all created automatically — none are checked into git)

| DB file | Created by | Contents |
|---|---|---|
| `bse_equity.db` | `market_announcements.py` | BSE Equity announcements (+ the idea-board tables added by the pipeline) |
| `bse_sme.db` | `market_announcements.py` | BSE SME announcements + corporate actions |
| `nse_equity.db` | `market_announcements.py` | NSE Equity announcements |
| `nse_sme.db` | `market_announcements.py` | NSE SME announcements |
| `market_users.db` | `bse_market_suite.py` (on first run) | Dashboard login accounts |

None of these ship in the repo — `.gitignore` excludes `*.db` on purpose, since they're either scraped data or local login credentials. Everything is regenerated by running the scripts.

## Quick start (one command)

```bash
git clone <your-repo-url> market-announcements-suite
cd market-announcements-suite
./quickstart.sh
```

This creates a virtualenv, installs everything in `requirements.txt`, builds a small **synthetic demo dataset** (20 sample announcements — no internet or scraping needed), scores it, and launches the dashboard at `http://localhost:8501`.

**Login:** `admin` / `admin@123` (you'll be prompted to change it — do that first, via the "Reset Password" tab).

To wipe the demo and start over: delete `bse_equity.db` and re-run `./quickstart.sh`.

## Manual setup (step by step)

If you'd rather not use the script, or you're on Windows:

```bash
# 1. Create and activate a virtual environment
python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. (Optional) set your Anthropic API key, only needed for the AI Digest tab
export ANTHROPIC_API_KEY=sk-ant-...     # Windows: set ANTHROPIC_API_KEY=sk-ant-...

# 4. Get data — pick ONE:
#    a) Real data — scrape today's announcements from all 4 sources:
python3 market_announcements.py fetch
#    b) Demo data — synthetic sample, no internet required:
python3 announcement_ideas_pipeline.py --db bse_equity.db demo

# 5. Score BSE Equity announcements for the Idea Board
python3 announcement_ideas_pipeline.py --db bse_equity.db run

# 6. Launch the dashboard
streamlit run bse_market_suite.py
```

## 1. Scraping data — `market_announcements.py`

```bash
# Fetch today's announcements from all 4 sources
python3 market_announcements.py fetch

# Fetch a specific date range
python3 market_announcements.py fetch --from 01-06-2026 --to 28-06-2026

# Fetch just one source
python3 market_announcements.py fetch --source bse_equity
python3 market_announcements.py fetch --source nse_sme

# BSE Equity watchlist mode (specific symbols only)
python3 market_announcements.py fetch --source bse_equity --symbols TCS INFY ITC

# Query what's already stored
python3 market_announcements.py query --source bse_equity --from 01-06-2026 --to 28-06-2026
python3 market_announcements.py query --source bse_sme --scrip ABCINFRA
python3 market_announcements.py query --source nse_equity --output results.json

# Text reports
python3 market_announcements.py report --from 01-06-2026 --to 28-06-2026

# DB statistics
python3 market_announcements.py stats
python3 market_announcements.py stats --source bse_sme

# Export to CSV
python3 market_announcements.py export out.csv --source nse_equity

# Unattended scheduled fetching (for cron / Task Scheduler)
python3 market_announcements.py schedule --every hourly
```

Cron example (all sources, every 3 hours):
```bash
0 */3 * * *  cd /path/to/market-announcements-suite && ./.venv/bin/python market_announcements.py fetch >> cron.log 2>&1
```

## 2. Scoring ideas — `announcement_ideas_pipeline.py`

Works on `bse_equity.db` only (the other 3 DBs aren't scored — the idea
taxonomy is BSE-Equity-specific in this build).

```bash
# Full pipeline: migrate schema + seed taxonomy + score everything new
python3 announcement_ideas_pipeline.py --db bse_equity.db run

# Rescore everything from scratch (e.g. after editing idea_rules.py)
python3 announcement_ideas_pipeline.py --db bse_equity.db run --recompute

# Only score a date range
python3 announcement_ideas_pipeline.py --db bse_equity.db run --from 2026-06-01 --to 2026-06-30

# Individual steps
python3 announcement_ideas_pipeline.py --db bse_equity.db migrate
python3 announcement_ideas_pipeline.py --db bse_equity.db seed
python3 announcement_ideas_pipeline.py --db bse_equity.db classify

# Build the synthetic demo DB (no real data needed)
python3 announcement_ideas_pipeline.py --db bse_equity.db demo
```

## 3. Dashboard — `bse_market_suite.py`

```bash
streamlit run bse_market_suite.py
```

The app reads the DBs **read-only** — run the two scripts above
separately (or on a schedule) to keep the data fresh; no need to restart
the dashboard when new data lands.

Pages (sidebar):
1. **Announcements** — search/filter per source, clickable PDF links. BSE Equity has an extra "Idea Board" sub-tab showing the categorized/scored ideas from step 2.
2. **Charts** — daily volume, category breakdown, timeline per source.
3. **Insights** — keyword frequency, trigger-keyword flags, topic clusters, and an optional Claude-generated digest (needs `ANTHROPIC_API_KEY`).
4. **My Activity** — your own view/search history for the session.

**Default login:** `admin` / `admin@123` — change it via the "Reset Password" tab after logging in, and use "Create User" (admin-only) to add teammates. Accounts live in `market_users.db`, created automatically on first run.

## Troubleshooting

- **"ModuleNotFoundError: idea_rules"** — you're missing `idea_rules.py`; make sure it's in the same folder as `announcement_ideas_pipeline.py` (it's included in this repo).
- **"Run: pip install streamlit-option-menu" / "...streamlit-authenticator"** — `pip install -r requirements.txt` wasn't run, or you're not inside the venv (`source .venv/bin/activate`).
- **AI Digest button errors** — `ANTHROPIC_API_KEY` isn't set. Everything else in the dashboard works fine without it.
- **Dashboard shows no data** — you haven't run `market_announcements.py fetch` (or the `demo` command) yet, or the `.db` files aren't in the same folder as `bse_market_suite.py`. Edit `DB_PATHS` near the top of `bse_market_suite.py` if you keep them elsewhere.
- **BSE SME / NSE fetching fails** — these hit live, undocumented HTML/JSON endpoints (`bsesme.com`, NSE's API) that can rate-limit, change shape, or require standard browser-like headers already set in the script. Re-running usually works; persistent failures likely mean the site changed and the scraper needs a small fix.
